# `auth`

[README](../README.md) › `auth`

Sign in, inspect identity, and manage API keys

## Subcommands

- [host](#command-auth-host) — Manage configured hosts
  - [alias](#command-auth-host-alias) — Manage host aliases
    - [add](#command-auth-host-alias-add) — Add an alias to a host
    - [list](#command-auth-host-alias-list) — List aliases
    - [remove](#command-auth-host-alias-remove) — Remove an alias from a host
  - [delete](#command-auth-host-delete) — Remove a host and all its profiles
  - [list](#command-auth-host-list) — List all configured hosts
  - [rename](#command-auth-host-rename) — Rename a host key
  - [use](#command-auth-host-use) — Set the default host
- [keys](#command-auth-keys) — Manage API keys [[admin]]
  - [create](#command-auth-keys-create) — Create an API key
  - [delete](#command-auth-keys-delete) — Revoke an API key
  - [list](#command-auth-keys-list) — List API keys
- [login](#command-auth-login) — Sign in with username and password
- [logout](#command-auth-logout) — Remove stored credentials and end the current session
- [profiles](#command-auth-profiles) — Manage saved server profiles
  - [delete](#command-auth-profiles-delete) — Delete a profile
  - [list](#command-auth-profiles-list) — List all hosts and profiles
  - [rename](#command-auth-profiles-rename) — Rename a profile
  - [show](#command-auth-profiles-show) — Show the resolved profile for the current context
  - [use](#command-auth-profiles-use) — Set the default profile for a host
- [quick](#command-auth-quick) — Quick Connect flows
  - [approve](#command-auth-quick-approve) — Approve a pending Quick Connect code
  - [login](#command-auth-quick-login) — Start a Quick Connect login flow
  - [status](#command-auth-quick-status) — Check whether Quick Connect is enabled
- [users](#command-auth-users) — List public users shown on the login screen
- [whoami](#command-auth-whoami) — Show the authenticated user and current server

## Examples

- `auth login --username demo --password secret`
